# FOMO — AI热点搜索 Chrome 浏览器扩展 项目全景描述

---

## 一、项目目的与顶层架构

FOMO 是一款 Chrome 浏览器扩展，核心目标是让用户一键启动一个智能搜索代理（Agent），由该代理自主在互联网上浏览和采集当前最新、最火的人工智能相关资讯，包括热门开源项目（如 GitHub Trending 上的 LLM、Agent、RAG 类项目）、重要学术论文（来自 ArXiv 等平台）、以及 AI 领域意见领袖在社交媒体上的讨论和推荐。代理的目标是自动收集约二十条不重复的高质量结果，收集完毕后用户可以将结果和执行日志下载为本地文件。

整体架构遵循 Chrome 扩展 Manifest V3 的标准模式，分为三个运行层次：弹出窗口（Popup）负责用户交互界面，后台服务工作线程（Background Service Worker）承载核心业务逻辑和 Agent 执行，两者之间通过 Chrome 扩展消息通信机制交互。Agent 的"大脑"是一个大语言模型（通过阿里云百炼平台的通义千问模型提供），Agent 通过调用浏览器操控工具来模拟用户在网页上的浏览行为，形成一个"思考—行动—观察"的反应式循环（ReAct 循环）。

## 二、技术选型与构建体系

### 编程语言

整个项目使用 TypeScript 编写，编译目标为 ES2022，启用严格模式。类型系统中引入了浏览器 DOM 类型定义、Chrome 扩展类型定义、Vite 客户端类型定义以及 Node.js 类型定义。模块系统采用 ESNext 模块规范，模块解析策略设为 bundler 模式以配合 Vite 的打包行为。项目不输出声明文件，也不生成声明映射。

### 构建工具

使用 Vite 5.x 作为构建工具。构建时采用多入口配置，一个入口是弹出窗口的 HTML 页面，另一个入口是后台脚本的 TypeScript 文件。构建产物输出到项目根目录下的 dist 文件夹，每次构建前会清空该目录。构建目标为 esnext，不进行代码压缩以方便调试，同时生成源码映射文件。

Vite 配置中设置了路径别名，将 at 符号映射到项目根目录。模块解析优先使用浏览器兼容的条件导出（browser、module、import、default），主字段优先级为 browser、module、main。

Rollup 的外部化配置将 chrome 全局对象和所有以 node 冒号开头的内置模块标记为外部依赖，不打包进产物。

输出文件的命名规则是：后台脚本固定输出为 background 目录下的 index.js；弹出窗口的入口脚本以入口名命名放在同名目录下；共享的代码块放在 chunks 目录下并带有哈希值；CSS 文件固定输出为 popup 目录下的 popup.css；其他资源文件放在 assets 目录下并带有哈希值。

Vite 配置中还使用了一个自定义插件来处理构建后的文件复制工作。该插件在构建束关闭时执行以下操作：确保 dist 目录存在，将项目根目录的 manifest.json 复制到 dist 目录，将 popup 目录下的 CSS 文件复制到 dist 的 popup 目录，创建 dist 的 icons 目录并将源 icons 目录下所有 PNG 和 SVG 格式的图标文件复制过去。

### 环境变量

构建时通过 Vite 的 loadEnv 函数加载环境变量，然后通过 define 配置将三个关键变量在编译时静态替换注入到代码中。这三个变量分别是：大模型 API 密钥、API 基础地址、以及使用的模型名称。如果环境变量未设置，API 密钥默认为空字符串，基础地址默认指向阿里云百炼平台的 OpenAI 兼容接口，模型名称默认为 qwen-max。用户应在项目根目录创建 .env.local 文件来配置这些值。

### 依赖关系

运行时依赖仅有 openai 官方 SDK（4.28 版本以上），用作与大模型 API 通信的客户端。开发依赖包括 Chrome 扩展类型定义、Node.js 类型定义、TypeScript 编译器（5.3 版本以上）以及 Vite 构建工具。

### 开发与构建命令

项目提供三个 npm 脚本命令：dev 命令使用 Vite 的构建监听模式进行开发，build 命令执行一次性生产构建，preview 命令用于预览构建产物。项目的包类型声明为 ESModule。

## 三、目录结构与文件命名约定

项目的根目录名为 fomo，其下的结构组织如下：

根目录层面放置项目配置文件，包括包管理器配置文件、TypeScript 编译器配置文件、Vite 构建配置文件、Chrome 扩展清单文件。根目录还有一个类型定义文件，作为整个项目的共享类型模块。此外还有一个图标资源目录，存放不同尺寸的扩展图标。

background 目录是后台服务工作线程的代码区域，其入口文件位于该目录的顶层。该目录下进一步划分为三个子目录：agent 子目录包含智能代理的核心逻辑，包括执行器、提示词模板和工具定义三个文件；browser 子目录包含浏览器操控层，有上下文管理和页面控制两个文件；llm 子目录包含大模型客户端的封装，目前只有阿里云百炼平台的客户端实现一个文件。

popup 目录是弹出窗口的代码区域，包含 HTML 入口页面、CSS 样式文件和 TypeScript 交互逻辑文件。

文件命名遵循以下约定：目录名使用全小写英文单词；TypeScript 文件使用全小写加点号分隔的命名方式（如 context.ts、dashscope.ts）；入口文件统一命名为 index 加对应扩展名；类型文件单独提取到根目录层面以便跨模块共享。

## 四、各模块职责与接口

### 共享类型模块（根目录 types.ts）

该文件定义了整个项目中所有模块共用的数据结构接口和类型别名。

搜索结果接口定义了一条 AI 资讯的完整数据结构，包含名称（标题）、信息来源地址、可选的产品主页地址、可选的 GitHub 仓库地址、可选的 ArXiv 论文地址、创新点描述、公司或团队名称、热度趋势评级（高中低三个级别）、标签数组、来源平台名称、以及发现时间戳。

Agent 状态接口包含已收集的结果数组、已访问链接集合（使用 Set 数据结构保证唯一性）、当前步骤编号、最大步骤数、目标收集数量、以及完成标志。

执行状态是一个字符串联合类型，涵盖空闲、运行中、暂停、已完成和错误五种状态。

状态更新接口包含执行状态、当前进度数值、总量数值、可选的当前操作描述和可选的错误信息。

日志条目接口定义了执行过程中每一步记录的结构，包含时间戳、日志类型（助手消息、工具调用、工具响应、用户消息、系统事件、错误共六种）、步骤编号和内容体。

完整执行日志接口包含任务开始时间、可选的结束时间、总步骤数、结果数量和日志条目数组。

消息类型定义了弹出窗口和后台之间通信的所有消息结构，使用可区分联合类型实现，通过 type 字段区分消息种类。

### Chrome 扩展清单（manifest.json）

清单版本为 3，是 Chrome 扩展的最新标准。扩展名称为"FOMO - AI热点搜索"，描述为一键搜索 AI 相关热门开源项目和论文。版本号为 1.0.0。

扩展图标提供三种尺寸：16像素、48像素和128像素，分别存放在 icons 目录下对应命名的 PNG 文件中。

权限声明包括：tabs 权限用于操控和查询标签页信息，scripting 权限用于在网页中注入和执行脚本，storage 权限用于扩展数据存储（虽然当前代码中未使用），downloads 权限用于下载结果文件，activeTab 权限用于获取当前活动标签页的临时访问权。

主机权限设置为通配符匹配所有 URL，允许代理访问任意网站。

后台脚本声明为模块类型的服务工作线程，入口为 background 目录下的 index.js 文件。

扩展的浏览器操作配置了默认弹出窗口为 popup 目录下的 index.html。

### 后台入口模块（background/index.ts）

这是后台服务工作线程的入口点，负责协调整个扩展的运行逻辑。

模块维护了几个模块级变量来跟踪全局状态：当前执行器实例（可为空）、当前结果数组、当前执行状态和当前执行日志（可为空）。

提供了三个向弹出窗口发送消息的辅助函数：发送状态更新、发送任务完成消息和发送错误消息。这三个函数都通过 Chrome 运行时消息接口发送消息，并静默捕获发送失败的错误（因为弹出窗口可能未打开）。

启动任务的函数接收一个标签页 ID 参数。函数首先检查是否已有任务在运行，如有则直接返回。然后获取浏览器上下文单例，设置当前标签页并尝试连接。连接成功后创建新的 Agent 执行器实例（默认目标为二十条结果、最大一百步），注册状态回调，运行执行器并等待结果。运行完毕后获取执行日志，将状态标记为已完成并通知弹出窗口。如果过程中出现任何错误，状态标记为错误并发送错误消息。无论成功还是失败，最终都会清理浏览器上下文并置空执行器引用。

停止任务的函数调用执行器的取消方法，置空引用并恢复空闲状态，同时清理浏览器上下文。

获取状态的函数返回一个包含当前执行状态、已收集结果数量和目标总量的对象。

模块注册了三个 Chrome 事件监听器。消息监听器处理来自弹出窗口的五种消息：启动任务（异步执行后立即响应成功）、停止任务、获取状态、下载结果（返回缓存的结果数组）、下载日志（返回执行日志对象）。对于未知消息类型返回错误信息。监听器返回 true 以保持消息通道异步开启。

标签页移除监听器在用户关闭标签页时从浏览器上下文中移除对应的页面连接。

扩展安装或更新监听器仅做日志记录。

### 浏览器上下文管理模块（background/browser/context.ts）

该模块提供了浏览器上下文管理器类，封装了对多个标签页连接的管理。

管理器维护两个内部状态：当前活动的标签页 ID 和一个以标签页 ID 为键、页面控制器为值的映射表。

获取或创建页面的私有方法首先在映射表中查找已有实例，如果没有则创建新的页面控制器实例。

连接页面方法接收一个页面控制器实例，如果该标签页已有连接则直接返回成功，否则调用页面控制器的连接方法，成功后将其加入映射表。

获取当前页面方法检查是否有设置当前标签页 ID，没有则返回空。如果映射表中没有对应的页面，会自动创建并连接。

切换标签页方法首先通过 Chrome API 激活目标标签页，然后获取或创建对应的页面控制器并更新当前标签页 ID。

导航方法有两条路径：如果当前页面已连接则通过页面控制器导航；否则使用 Chrome tabs API 更新标签页 URL，等待加载完成后再创建并连接新的页面控制器。

等待标签页加载的私有方法通过注册 Chrome 标签页更新事件监听器来实现，当目标标签页的状态变为完成时解除监听。同时设置了三十秒的超时保护，超时后无论是否加载完成都会解除等待。

清理方法遍历所有已连接的页面并逐一断开连接，清空映射表并重置当前标签页 ID。

移除特定页面的方法先断开连接再从映射表中删除，如果被移除的标签页恰好是当前活动标签页则重置当前标签页 ID。

整个模块以单例模式对外暴露浏览器上下文管理器实例，通过一个工厂函数获取，首次调用时创建，后续调用返回同一实例。

### 页面控制器模块（background/browser/page.ts）

该模块提供了浏览器页面控制器类，是对单个标签页操作的完整封装。所有浏览器操控都通过 Chrome 扩展的 scripting API 实现，而非使用 Chrome DevTools Protocol。因此连接和断开方法实际上是空操作——页面控制器始终处于"已连接"状态。

导航方法通过 Chrome tabs API 更新标签页的 URL，然后等待页面加载完成。等待加载的实现同样使用标签页更新事件监听器，收到完成状态后额外等待五百毫秒以确保页面渲染就绪，同时设有三十秒超时保护。

获取当前 URL 和页面标题的方法通过 Chrome tabs API 直接查询标签页信息。

私有的脚本执行方法封装了 Chrome scripting API 的 executeScript 调用，将要执行的函数注入到目标标签页中运行，返回执行结果。错误时记录日志并返回空值。

点击元素方法向标签页注入一段脚本，该脚本根据 CSS 选择器查找目标元素，找到后调用其点击方法。如果找不到元素，返回包含选择器信息的错误消息。点击成功后等待五百毫秒让页面响应。

文本输入方法同样注入脚本，定位输入框元素后设置焦点，直接赋值文本内容，然后触发 input 和 change 两个事件以确保页面的响应式框架能捕获到值变化。

按键方法注入脚本构造键盘事件并分发给当前活动元素。对于回车键有特殊处理——会额外查找最近的表单元素并触发提交事件。按键后等待五百毫秒。

滚动方法注入脚本，向下滚动五百像素或向上滚动五百像素，滚动后等待三百毫秒。

获取页面文本方法是最复杂的注入脚本操作。它先克隆整个 body 元素（避免修改原页面），移除所有 script、style 和 noscript 标签，然后使用 TreeWalker 遍历所有文本节点，收集非空文本内容并用空格拼接。最终结果截断到一万五千字符以内。

获取页面链接方法注入脚本，查询所有带 href 属性的锚点元素，提取每个元素的文本内容（截断到一百字符）和 href 属性值，过滤掉缺少文本或链接的项。

等待特定时长的方法基于 Promise 包装的 setTimeout 实现。

等待选择器出现的方法采用轮询策略，每五百毫秒检查一次目标选择器对应的元素是否存在于页面 DOM 中，直到找到或超时（默认一万毫秒）。

所有与标签页交互的方法都包含了完善的错误捕获，返回中文描述的成功或失败消息。

### 大模型客户端模块（background/llm/dashscope.ts）

该模块封装了与阿里云百炼平台的大模型 API 交互逻辑。

客户端类在构造时读取三个通过 Vite 环境变量注入的配置值：API 密钥、基础 URL 和模型名称。如果 API 密钥为空，构造过程会抛出错误并提示用户在 .env.local 文件中配置。

客户端底层使用 OpenAI 官方 SDK 创建实例，将 API 密钥和基础 URL 传入，并启用了"允许在浏览器环境中运行"的选项（因为 Chrome 扩展的后台脚本虽然不是传统网页但仍属于浏览器环境）。

聊天方法接受消息数组和可选的工具定义数组作为参数，调用 SDK 的聊天补全接口。如果提供了工具列表，工具选择模式设为自动（由模型自行决定是否调用工具）。返回完整的聊天补全响应对象。

简单文本补全方法接受单个提示文本，内部将其包装为用户消息发送给模型，返回模型回复的文本内容。

模块同样以单例模式暴露客户端实例。

### Agent 提示词模块（background/agent/prompts.ts）

该模块负责生成 Agent 的系统提示词和初始用户消息，所有提示词内容动态注入当前日期信息以确保搜索结果的时效性。

内部辅助函数获取当前日期信息，返回年、月、日的数值，格式化的中文日期字符串，以及月份的英文名称。

系统提示词将 Agent 定义为专业的 AI 资讯搜索代理，明确告知当前日期。提示词中设定了收集十到二十条高质量 AI 相关资讯的目标，并详细列举了五类搜索策略：Twitter/X 平台（列出了 Karpathy、Andrew Ng、Yann LeCun 等知名 AI 研究者的主页）、GitHub Trending 页面（筛选 Python 和 TypeScript 语言的项目）、Google 搜索（提供了包含当前年月的示例搜索词）、ArXiv 学术论文平台（关注 AI、计算语言学和机器学习分类）、以及 Reddit 的机器学习和本地大模型社区。

提示词还详细说明了每个可用工具的名称和用途，列出了注意事项（如立即保存发现、避免重复、应对页面加载问题、灵活切换策略），以及期望的输出格式。

初始用户消息指示 Agent 从 Twitter/X 平台的 Karpathy 主页开始搜索，明确目标数量和当前日期，并要求立即导航到指定 URL。

模块同时导出一个预生成的系统提示词常量以兼容旧代码引用。

### Agent 工具定义与执行模块（background/agent/tools.ts）

该模块分为两部分：工具定义和工具执行器。

工具定义是一个符合 OpenAI 聊天补全工具格式的数组，共定义了十一个工具。每个工具都有名称、描述和参数模式定义。这些工具分为三类：

浏览器操控类工具包括：导航（必需参数为 URL 字符串）、点击（必需参数为 CSS 选择器）、文本输入（必需参数为选择器和文本内容）、按键（必需参数为键名）、滚动（必需参数为方向枚举值）、获取页面内容（无参数）、获取链接列表（无参数）、等待（必需参数为一到十之间的秒数）。

数据管理类工具包括：保存结果（必需参数为名称、来源 URL、创新点描述和来源平台，可选参数为产品 URL、GitHub 链接、ArXiv 链接、公司名、热度趋势评级、标签数组）、获取当前状态（无参数）。

流程控制类工具：完成任务（必需参数为完成原因字符串）。

工具执行器类接收 Agent 状态引用进行构造。执行方法接收工具名和参数对象，通过开关分支路由到对应的处理逻辑。

对于浏览器操控类工具，执行器获取当前页面控制器实例并调用其相应方法。在获取页面内容时，执行器会同时获取页面标题和 URL 一并返回，并将内容截断到八千字符。获取链接时限制显示前五十条。

保存结果的逻辑首先将传入参数组装为完整的搜索结果对象（缺失字段使用默认值，如公司默认为"Community"，热度默认为"Medium"，标签默认为包含"AI"的数组，发现时间为当前时间戳）。然后检查重复性——如果新结果的名称（不区分大小写）、来源 URL 或 GitHub 链接与已有结果匹配，则跳过并返回重复提示。去重通过后将结果加入状态的结果数组。如果结果数量达到目标值，设置完成标志并返回庆祝消息。

获取状态方法返回格式化的进度信息，包括已保存数量与目标的比值、当前步骤与上限的比值、完成状态以及所有已保存结果的编号列表。

完成工具直接将状态标记为完成并返回完成信息。

### Agent 执行器模块（background/agent/executor.ts）

这是整个项目的核心模块，实现了 ReAct（Reason + Act）循环的完整执行引擎。

执行器类在构造时初始化 Agent 状态（默认目标二十条、最大一百步）、消息历史（初始包含系统提示词和初始用户消息两条）、工具执行器实例和执行日志。执行日志在初始化时记录开始时间，并将系统消息和初始用户消息各作为一条日志条目记录。

状态回调机制允许外部注册一个回调函数，执行器在关键节点（初始化、每次工具执行、完成等）调用该回调，传递包含当前进度、状态和操作描述的更新对象。

取消方法设置取消标志并将完成状态标记为真，同时记录取消事件日志。

主循环方法是一个异步函数，首先获取大模型客户端单例，然后进入循环。循环的继续条件是：未完成、未取消、且当前步骤未超过最大步骤上限。

每个循环迭代中：递增步骤计数器，调用大模型的聊天接口传入当前消息历史和工具定义，获取模型响应。如果响应为空则记录错误并跳过。

如果模型回复包含文本内容（思考过程），则截取前两百字符记录到控制台并截取前一百字符作为状态更新发送。

模型的助手消息（包括文本内容和工具调用信息）被记录到日志并追加到消息历史。

如果模型请求了工具调用，则依次处理每个工具调用：解析工具名和参数（参数解析失败时使用空对象），记录调用日志，通过工具执行器执行工具，记录执行结果日志（结果超过五千字符会截断），最后将工具执行结果以 tool 角色追加到消息历史。

如果模型的结束原因为 stop 且没有工具调用（即模型试图结束对话），执行器会检查结果数量。如果未达标，则追加一条用户消息催促模型继续搜索或使用 finish 工具说明原因。如果已达标则标记完成。

消息历史控制是一个重要的边界处理：当消息数量超过五十条时，保留第一条系统消息和最近四十条消息，丢弃中间的旧消息以防止超出模型的上下文窗口限制。

每步的错误处理会捕获异常并将错误信息作为用户消息追加到历史中，提示模型尝试其他方法继续搜索。

循环结束后记录结束时间、总步骤数和结果数量到执行日志，返回收集到的所有结果。

### 弹出窗口界面模块（popup/）

弹出窗口由 HTML 结构、CSS 样式和 TypeScript 交互逻辑三个文件组成。

HTML 结构定义了一个紧凑的界面布局：顶部是品牌标志"FOMO"和一个圆形状态指示灯；中央是一个 SVG 实现的环形进度条，内部显示当前进度数字和目标总量；下方是一行当前操作描述文本；底部是四个圆形控制按钮——开始（播放图标）、停止（方块图标）、下载结果（下载箭头图标）和下载日志（文档图标）；最底部是可隐藏的错误提示区域。

CSS 样式采用了简洁现代的设计语言。定义了 CSS 自定义属性作为设计令牌，包括白色主背景、浅灰次背景、绿色强调色（用于激活状态和进度指示）、主文本色和次文本色、红色用于错误和停止、琥珀色用于日志按钮。

弹出窗口固定宽度为二百四十像素，最小高度二百八十像素。容器采用弹性盒纵向布局，元素之间间距十六像素，水平居中对齐。

状态指示灯有四种视觉状态：空闲时为灰色圆点、运行时为带绿色辉光和脉动动画的圆点、完成时为静态绿色圆点、错误时为红色圆点。脉动动画通过关键帧定义，在透明度零点六到一和缩放一到一点一之间交替。

环形进度条通过 SVG 圆形元素实现，背景环为浅灰色描边，前景环为绿色描边带圆角端点。进度变化通过过渡动画的描边偏移量控制，过渡时长零点五秒。

控制按钮为三十六像素的圆形按钮，使用弹性盒居中对齐图标。悬停时微放大至一点零五倍，点击时缩至零点九五倍。禁用状态透明度降至零点三且禁止光标点击。开始按钮激活时为绿色背景白色图标，停止按钮激活时为红色背景，下载按钮激活时为绿色边框，日志按钮激活时为琥珀色边框。

TypeScript 交互逻辑获取所有 DOM 元素引用后定义了几个核心函数。

更新进度环函数根据当前进度值计算描边偏移量（环的周长为三百二十七像素，基于半径五十二像素计算），更新 SVG 元素的样式属性和数字显示。

更新界面状态函数根据传入的状态更新对象，切换状态指示灯样式、更新进度、更新操作描述文本、以及根据不同状态（运行中、已完成、错误、空闲）调整四个控制按钮的禁用状态。运行中时只有停止按钮可用；完成时开始、下载和日志按钮都可用；错误时开始按钮可用，下载按钮取决于是否有缓存结果，日志按钮取决于是否有日志；空闲时与错误状态相似。

下载结果函数将缓存的搜索结果数组序列化为格式化的 JSON 字符串，创建 Blob 对象和对象 URL，通过 Chrome 下载 API 触发保存对话框。文件名包含 fomo-results 前缀和时间戳。

下载日志函数先向后台发送下载日志消息获取日志对象，然后以相同方式创建 JSON 文件并触发下载，文件名前缀为 fomo-logs。

启动任务函数首先查询当前活动标签页，验证标签页存在且不是 Chrome 内部页面（chrome:// 或 chrome-extension:// 开头的 URL 不允许运行），然后重置缓存和进度显示，向后台发送启动消息并立即更新界面为运行状态。

停止任务函数向后台发送停止消息并更新界面为停止状态。

模块注册了来自后台的消息监听器，处理三种消息：状态更新（直接更新界面）、任务完成（缓存结果并更新界面为完成状态）、错误（更新界面为错误状态）。

初始化时向后台发送获取状态消息以同步当前状态（处理弹出窗口关闭后重新打开的场景），并绑定四个按钮的点击事件。

## 五、模块间数据流

整体数据流形成一个以后台服务工作线程为中心的星形拓扑：

用户点击弹出窗口的开始按钮时，弹出窗口发送包含当前标签页 ID 的启动消息到后台。后台收到消息后，通过浏览器上下文管理器连接到指定标签页，创建 Agent 执行器实例并启动主循环。

在主循环中，Agent 执行器将消息历史发送给大模型客户端，客户端通过 OpenAI 兼容接口将请求转发到阿里云百炼平台。大模型返回包含思考过程和工具调用指令的响应。

执行器解析工具调用指令，通过工具执行器路由到具体的处理逻辑。对于浏览器操控类工具，工具执行器调用浏览器上下文管理器获取当前页面控制器，页面控制器通过 Chrome scripting API 向目标标签页注入脚本执行操作。脚本执行结果沿原路返回给工具执行器，工具执行器将结果格式化后返回给 Agent 执行器，执行器将其追加到消息历史中供大模型参考。

对于保存结果类工具调用，工具执行器直接操作 Agent 状态对象，将去重后的搜索结果加入结果数组。

每次状态变化时，执行器通过回调函数将进度信息传回后台入口模块，后台再通过 Chrome 运行时消息接口将状态更新推送到弹出窗口。弹出窗口收到更新后刷新进度环、状态灯和操作描述。

任务完成后，完整的结果数组被缓存在后台和弹出窗口两处。用户点击下载按钮时，弹出窗口直接使用本地缓存（对于结果）或向后台请求（对于日志）获取数据，然后通过浏览器下载功能保存为本地 JSON 文件。

## 六、关键业务逻辑

### ReAct 循环决策引擎

整个 Agent 的核心是一个有上限的 ReAct 循环。每一轮循环中，大模型基于完整的对话历史（包括系统指令、用户请求、自身之前的思考和工具调用结果）做出下一步决策：选择调用哪个工具、传入什么参数。工具执行后的结果反馈给模型，模型据此决定继续搜索还是完成任务。

这种设计的关键在于模型拥有对搜索策略的完全自主权——它可以自由选择先访问哪个网站、搜索什么关键词、如何在页面间导航。提示词提供了策略建议但不强制执行顺序。

### 结果去重算法

去重基于三个维度的任一匹配：名称的小写形式完全相等、来源 URL 完全相等、或 GitHub 链接完全相等（后两者仅在非空时比较）。这种策略可以有效防止同一项目通过不同渠道被重复收集。

### 消息历史压缩

当对话消息超过五十条时，执行器保留系统提示词（第一条消息）和最近四十条消息，丢弃中间的历史消息。这是一种滑动窗口策略，在保留关键上下文（系统指令和近期交互）的同时控制模型的输入长度，避免超出 token 限制。

### 未达标继续催促机制

当模型试图结束对话（返回 stop 且无工具调用）但结果数量未达标时，执行器会自动注入一条用户消息，告知模型当前进度不足并要求继续搜索或给出无法继续的理由。这确保了模型不会过早放弃。

### 页面内容提取策略

页面文本提取采用"克隆—清理—遍历"三步法：先克隆 DOM 避免影响原页面，移除脚本和样式元素消除噪音，然后使用 TreeWalker 只遍历文本节点收集有意义的文本内容。结果截断到一万五千字符以平衡信息量和传输效率。

## 七、命令行接口与操作选项

该项目不是命令行工具而是浏览器扩展，因此没有传统意义上的 CLI 参数。与用户的交互通过弹出窗口的四个按钮完成：

开始按钮：在当前活动标签页上启动 AI 资讯搜索代理。前提条件是当前页面不能是 Chrome 内部页面。

停止按钮：终止正在运行的搜索任务。仅在任务运行期间可用。

下载结果按钮：将搜索到的 AI 资讯导出为 JSON 文件。仅在有结果时可用。

下载日志按钮：将完整的执行日志（包含每一步的模型思考、工具调用和工具响应）导出为 JSON 文件。仅在任务完成或出错后有日志时可用。

开发者通过 npm 脚本操作项目：dev 用于开发模式（文件变更时自动重新构建），build 用于生产构建。构建产物需要在 Chrome 浏览器的扩展管理页面以开发者模式加载 dist 目录来安装。

## 八、错误处理策略

项目采用多层错误处理策略：

在弹出窗口层面，启动任务时的错误（如无法获取标签页、标签页为内部页面）被捕获并直接显示在错误提示区域。

在后台入口层面，整个任务执行流程包裹在 try-catch-finally 块中。catch 块将错误状态和消息推送到弹出窗口显示。finally 块确保无论成功还是失败都会执行浏览器上下文清理。

在 Agent 执行器层面，每个循环步骤都有独立的 try-catch。捕获到的错误不会终止整个循环，而是被转化为用户消息注入对话历史，提示模型尝试其他方法继续搜索。这使得 Agent 具有自我恢复能力。

在页面控制器层面，所有 Chrome scripting API 调用都包含错误捕获，返回中文描述的失败消息而非抛出异常。这些消息会作为工具执行结果传回给模型，模型可以据此调整策略。

在消息通信层面，向弹出窗口发送消息时的错误被静默捕获（因为弹出窗口可能未打开），不影响后台逻辑运行。

大模型客户端在 API 密钥缺失时会在构造阶段直接抛出错误，提前终止流程并给出明确的配置提示。

工具参数解析失败时使用空对象作为降级处理，确保工具执行不会因为参数格式问题而完全中断。
